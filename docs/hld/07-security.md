# 07-security.md — STRIDE Threat Model

This document outlines the security architecture for `explorers-cli` v1.0. It defines the trust boundaries of the local host system and provides a STRIDE threat assessment with corresponding design mitigations.

## Trust Boundaries

The system is split across three trust zones:

1. **Host Workstation / OS Environment (Trusted)**: Local filesystem containing credentials, configuration variables, and data databases. Access is restricted to the operator.
2. **Minecraft Process Sandboxes (Semi-Trusted)**: Java child processes executing game logic. The server directories are accessible by the manager, but the game threads are exposed to player inputs.
3. **Public Network / Connected Clients (Untrusted)**: Public players sending chat messages and external LLM APIs processing payload packets.

## Top Residual Risks

1. **Minecraft Offline Mode Vulnerability (Spoofing)**: If the operator runs the Minecraft server with `online-mode=false`, any player can log in under an administrator's username (e.g. "Steve") and immediately invoke admin-level agents.
   - _Mitigation_: Warn the operator in installation documentation to run servers in online mode or secure the network gateway using proxy firewalls (e.g. BungeeCord).
2. **Plain-Text File Overwriting (Tampering)**: While the server is running, the agent is blocked from writing binary files (.nbt) or deleting directories. However, it can still corrupt plain-text game files (like `.json` configuration files) if they fall under the sandbox directory and the tools allow it.
   - _Mitigation_: Keep agent tool directories limited to specific sub-folders (like `plugins/my-agent/`) instead of the root directory where possible.
3. **LLM Hallucination of Commands**: An agent might attempt to chain commands or insert unexpected parameters that bypass basic allowlist filters if the allowlist prefixes are configured too loosely.
   - _Mitigation_: Keep allowed commands restricted to specific exact tokens (e.g., `whitelist add` rather than broad prefixes like `whitelist`).
4. **Shared Terminal Misuse**: Read-only mode prevents application-level mutations, but it cannot prevent a user with shell access from editing files or running other local commands outside `explorers-cli`.
   - _Mitigation_: Pair `--read-only` with host OS account controls, terminal kiosk restrictions, or limited shell access for shared-host deployments.

## Compliance Mapping

- **Local Storage / Privacy (GDPR-aligned)**: All session logs, user inputs, and player histories are stored locally inside the file-based SQLite database (`data/sessions.db`). Telemetry is strictly opt-in and excludes player usernames or chat content (NFR-PRV-001 / NFR-PRV-002).
- **EULA Compliance**: The operator holds sole responsibility for accepting the Minecraft EULA by populating `eula.txt` within the server directory (NFR-COMP-006).
